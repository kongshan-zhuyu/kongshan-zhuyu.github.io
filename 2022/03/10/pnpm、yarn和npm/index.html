一、什么是pnpm

含义：pnpm是一个包管理的工具

优势：

  • 安装包非常的快速
  • 节省磁盘空间，空间利用率高
  • 支持 monorepos
  • 安全性高，避免非法访问依赖的问题

二、特性详解

1、安装速度快

2、磁盘空间利用率高

  • 不会重复安装同一个包，对依赖（源文件在.pnpm-store）建立硬链接存储在node_modules下的.pnpm上，然后在node_modules下创建软链接（基于.pnpm下），而npm1/npm2会重复创建
  • 对于一个包的不同版本，pnpm会复用之前的代码，假设新版本的依赖多了一个文件，磁盘不会重写整个文件，仅仅写入新增的文件

3、支持 monorepos

pnpm命令的支持

例如：在根目录下 pnpm add A -r, 那么所有的 package 中都会被添加 A 这个依赖

4、安全性高

巧妙的包管理管事方式

原本存在的问题比如A依赖B，B依赖C，A中直接调用C里面不会有问题，像这种一个库使用了不属于其dependencies 里的Package 称之为 幻影依赖/幽灵依赖/隐式依赖

但是是可能存在问题的

问题1：当依赖B更新以后C的版本可能变了，也可能不需要C里，此时在A中引用C就会出现问题

问题2：在 monorepo 项目中，如果 A 依赖 X，B 依赖 X，还有一个 C，它不依赖 X，但它代码里面用到了 X。由于依赖提升的存在，npm/yarn 会把 X 放到根目录的 node_modules 中，这样 C 在本地是能够跑起来的，因为根据 node 的包加载机制，它能够加载到 monorepo 项目根目录下的 node_modules 中的 X。但试想一下，一旦 C 单独发包出去，用户单独安装 C，那么就找不到 X 了，执行到引用 X 的代码时就直接报错了

缺点：

生态圈还不够完善，可能会存在安装依赖缺失的问题
